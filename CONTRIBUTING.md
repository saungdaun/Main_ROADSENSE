# Contributing to Main_ROADSENSE

Thank you for your interest in contributing to Main_ROADSENSE.

This project focuses on digital pavement condition assessment, road distress classification, and field-based data collection using Android.

---

## How to Contribute

### 1. Fork the Repository
Fork this repository and clone it to your local machine.

### 2. Create a Branch
Create a feature branch from main.

Example:
git checkout -b feature-your-feature-name

### 3. Follow Coding Standards
- Use Kotlin for Android components
- Follow MVVM architecture (if applicable)
- Keep UI logic separated from domain logic
- Write clean, readable, and well-documented code

### 4. Commit Clearly
Use clear and descriptive commit messages.

Example:
feat: add road distress classification logic
fix: correct GPS accuracy calculation

### 5. Open a Pull Request
Submit a pull request to the main branch with:
- Clear explanation of the changes
- Screenshots (if UI changes)
- Description of testing performed

---

## Reporting Issues

If you find bugs or want to suggest improvements:

- Use the GitHub Issues tab
- Provide clear reproduction steps
- Include screenshots or logs if relevant

---

## Code Style

- Kotlin conventions
- Meaningful variable names
- Avoid hardcoded values
- Comment complex logic

---

## Scope of Contributions

We welcome contributions related to:

- Road distress detection logic
- Pavement scoring algorithms
- GPS and sensor integration
- UI/UX improvements
- Performance optimization
